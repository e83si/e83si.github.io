DSC Supply




Testing instructions for interface faults in the DSC
An interface fault is saved in the DSC control module. Details can be determined using 'control-module functions - Read fault memory'.

For the DSC function, information from partner control modules is essential. The fault is entered of the relevant partner control module (= interface to the DSC) sends a default value or invalid ID instead of the required information to the DSC and this impairs the DSC function. In this case, a fault memory entry should also be made in the corresponding partner control module.

For this fault, the cause does NOT lie in the DSC control module or in the DSC software!

Possible causes of faults:
- the relevant partner control module is not yet operational
- the sensor system of the relevant partner control module is defective
- general malfunction of the partner control module

Read the fault memory of the relevant partner control module and continue troubleshooting according to test plan in the relevant partner control module. Delete the fault in the DSC control module.

If work is currently being carried out on the vehicle and this causes the interface fault in the DSC, under certain circumstances the corresponding fault memory entry can only be made in the partner control module after a terminal change. If necessary, run a terminal change. If no fault is saved in the relevant partner control module, it can be assumed that there is a sporadic fault. There is no repair measure available for this case: Delete the fault in the DSC control module.

Note on clearing the fault memory
Under certain circumstances, the fault in the DSC can only be deleted after a terminal change. In this case, switch the ignition off, wait 10 seconds, switch the ignition on again and then delete the fault.
